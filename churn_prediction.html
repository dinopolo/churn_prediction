Prevendo o Churn utilizando Machine Learning¶

A taxa de churn, tambm chamado taxa de atrito ou rotatividade de cliente, a taxa na qual clientes para de fazer negcios com uma entidade, ou seja, a taxa de evaso dos clientes. Em servios por assinatura como Spotify, Netflix ou TV por assinatura, ela seria representada pelos clientes que cancelaram a assinatura em determinado perodo.

Uma empresa pode utilizar o churn para comparar com a taxa de crescimento e determinar se a empresa est tendo ganhos ou perdas. Alm disso, a melhor taxa de churn que ele no ocorra, no entanto quase impossvel que isso ocorra e muito difcil estimar um valor que seja verdadeiro para todos os tipos de negcios, mas, por exemplo, para SaaS o churn sugerido fica entre 5% e 7% ao ano.

Um questionamento importante a se fazer : quais so os motivos para o churn acontecer?

  • Entrega de valor menor do que foi prometido;
  • A reputao do produto est manchada;
  • Os clientes encontraram uma alternativa nos concorrentes;
  • Os clientes no necessitam ou no tm mais insteresse no produto;
  • Cliente encerrou a operao que necessitava do produto; e
  • Muitos outros motivos.

Mas como um modelo de Machine Learning pode auxiliar com o churn?

Construindo um modelo de Machine Learning utilizando dados dos clientes que pararam de fazer negcios na empresa, podemos prever quais clientes ainda fazem negcios, no entanto esto prestes a parar e, assim, tomar as medidas necessrias para que esses clientes continuem com a empresa.

Por isso, o objetivo desse projeto avaliar diferentes modelos de Machine Learning e analisar qual que apresenta os melhores resultados para prever o churn do conjunto de dados utilizado.

def string_to_float(entrada):
    """
    Converte um objeto do tipo string para float.
    
    # Argumentos:
        entrada: objeto string ou coluna de DataFrame do tipo string
        
    # Resultados:
        Valor convertido em float, se a variel permitir a converso.
        NaN, caso no seja possvel a converso.
    """
    
    try:
        return float(entrada)
    except ValueError:
        return np.nan

# Funo para classificar se  int ou float ou, se nenhum deles, retorna False
def int_or_float(argumento):
    """
    Argumento -> varivel para classificar.
    Se 'Argumento' for int ou float retorna True; se nenhum deles, retorna False.
    """
    if pd.api.types.is_integer_dtype(argumento):
        return True
    elif pd.api.types.is_float_dtype(argumento):
        return True
    else:
        return False

In [26]:
def val_modelo(clf, X, y, quite=False):
    """
    Realiza a validao cruzada com os dados de treino para determinado modelo.
    
    Argumentos:
        X: conjunto de dados, apenas com as variveis independentes;
        y: varivel objetivo;
        clf: modelo de classificao do Scikit-learn;
        quite: bool, indicando se a funo deve imprimir os resultados ou no.
        
    Return:
        float com a mdia dos scores da validao cruzada.
    """
    
    X = np.array(X)
    y = np.array(y)
    
    scores = cross_val_score(clf, X, y, scoring='roc_auc')
    
    if quite==False:
        print("AUC: {:.2f} (+/- {:.2f})".format(scores.mean(), scores.std()))
        
    return scores.mean()

Otimização dos parâmetros¶

O LightGBM possui diversos parmetros que podem ser ajustados, alguns afetam mais o desmepenho do modelo e outros menos. Uma boa abordagem definir uma taxa de aprendizado (learning_rate) e nmero de estimadores (n_estimators), fazer o ajuste de outros parmetros e, no fim, verificar outras taxas de aprendizado.

Irei comear ajustando o nmero de estimadores e para isso foi definido o learning_rate como sendo 0.1.

Aplicações¶

Mas e agora voc pode estar se perguntando:

Mas eu no trabalho em empresa que vende servios ou no precisa de assinatura para obter o produto da minha empresa?

Ou ainda

A empresa que trabalho ou meu negcio vende apenas para outras empresas (CNPJs) ou varejistas?

Para esses casos, diversas abordagens podem ser utilizados, vou citar apenas alguns exemplos:

  • No caso de venda de produtos finais, pode fazer uma anlise de quanto tempo um cliente no compra do seu produto ou na sua empresa atravs dos cadastros dos clientes ou dos clientes que informam CPF no momento da compra;
  • Para empresas que vendem produtos para CNPJs ou varejistas pode-se criar um questionrio para os clientes deles tendo como pergunta principal: "Voc consome o produto da marca XYZ?", mas lembrando que o questionrio deve ser criado de acordo com o produto vendido, porque, por exemplo, a renda familiar pode ser ou no um fator essencial para compra de determinado produto.

Ainda h diversas formas de se utilizar, cabe a pessoa que conhece o tipo de negcio da empresa e o cientista de dados definirem a melhor abordagem para desensolver um projeto nessa rea.
